# Technical Datasheet

## Environment Composition
- **Topology Size**: Procedurally generated environments consist of exactly 100 host array slots for tensor-shape stability. Active topologies range from 15-30 nodes. Unused nodes are padded with `169.252.0.0/16` addresses.
- **State Representation**: `GlobalNetworkState` (and its immutable `EnvState` equivalent) contains categorical host attributes, firewall ACLs, subnet topologies, and agent visibility masks. Categorical properties (e.g., privilege levels, decoy flags) are encoded against static codebooks documented in `netforge_rl/core/functional.py`.
- **Scenarios**: Two reference implementations provided at v2.0 (`ransomware`, `apt_espionage`).

## Metrics and Reward Signals
- Reward signals are decoupled from core mechanics and defined exclusively per-scenario via the `BaseScenario` class.
- Canonical scalar metrics emitted per episode include:
  - Mean Time To Containment (MTTC)
  - SLA Uptime
  - Total Exfiltrated Data Volume
  - Host Compromise/Isolation Counts

## Data Generation
- **Source**: Procedurally generated by `netforge_rl/topologies/network_generator.py`.
- **Determinism**: Fully deterministic based on the provided integer seed and `random.Random` instance.
- **Splits**: Leaderboard evaluation utilizes seeds 0 through 49. Training may occur on any seed outside this evaluation range.
- **Limitations**: Synthetic generation. Network traffic representations (`pcap_synthesizer`) are structurally accurate but are not backed by real-world telemetry captures.

## Supported Use Cases
- Reinforcement learning policy training (PPO, MAPPO).
- Foundation model zero-shot evaluation via the `Sim2RealBridge`.
- Large Language Model parameter-efficient fine-tuning (LoRA + PPO).
